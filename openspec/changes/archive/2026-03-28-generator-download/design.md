## Context

当前myproject-generator模块已实现代码生成功能，但生成的代码只能通过在线预览查看，用户需要手动复制代码到项目中，操作繁琐且容易出错。

## Goals / Non-Goals

**Goals:**
- 实现生成代码的ZIP打包下载功能
- 支持选择性下载（后端代码、前端代码）
- 下载文件按正确包路径目录结构组织
- 支持单文件预览和下载

**Non-Goals:**
- 不支持代码直接部署到服务器
- 不支持批量多次下载（暂只支持单次生成下载）

## Decisions

1. **ZIP打包方式选择** — 使用Java原生java.util.zip包实现，无需引入额外依赖，轻量且可靠。

2. **下载接口设计** — 采用POST /generator/download接口，接收生成请求参数，返回ZIP文件流。

3. **文件目录结构** - 后端代码按照com/example/包路径生成目录，前端代码按src目录结构生成。

4. **临时文件处理** - 使用完成后删除临时生成的源代码文件，避免磁盘空间浪费。

## Risks / Trade-offs

- [风险] 生成文件过大导致下载慢 → 缓解：ZIP压缩会减少文件体积
- [风险] 中文字符编码问题 → 缓解：统一使用UTF-8编码
- [风险] 临时文件未清理 → 缓解：使用finally块确保文件删除
